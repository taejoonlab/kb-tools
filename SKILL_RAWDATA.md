# Raw-data Accession Convention (RawDataAvailable)

## 개요
논문이 공개한 **원시 데이터(raw data) 저장소 accession**을 노트에 구조화해 기록하고,
논문 자신이 기탁한 데이터가 있으면 `RawDataAvailable` 태그를 붙이는 KB 공통 규약이다.
어느 KB에서든 동일하게 적용한다(주제 태그와 독립적인 **속성(attribute) 태그**).

대상 저장소와 accession 패턴:

| DB (라벨) | prefix 예 |
|-----------|-----------|
| GEO | `GSE#####`, `GSM######` |
| SRA | `PRJNA#####`, `SRP######`, `SRR######`, `SRX######` |
| ENA | `PRJEB####`, `ERP######`, `ERR######`, `ERX######` |
| DDBJ | `PRJDB###`, `DRA######`, `DRR######`, `DRX######` |
| ArrayExpress / BioStudies | `E-MTAB-####`, `E-GEOD-####` |
| PRIDE / ProteomeXchange | `PXD######` |
| MassIVE | `MSV#########` |

## 핵심 구분: self-deposited vs cited
accession이 본문에 등장한다고 모두 "이 논문의 raw data"는 아니다. **반드시 문맥으로 구분**한다.

- **self-deposited (본 연구 기탁)** → `raw_data:` frontmatter + `RawDataAvailable` 태그 대상
  - 신호구: "have been/were deposited (in) … under accession", "data generated in this study",
    "reported in this paper", "This paper / This study" (Key Resources의 Deposited Data 표),
    "Data (and materials) availability … accession", "submitted to … under accession"
- **cited (인용·재사용/외부 데이터)** → 본문 섹션에만 기재, 태그·frontmatter 대상 아님
  - 신호구: "downloaded from", "obtained from", "retrieved from", "publicly available",
    "previously published", "reported by X et al.", "(ref. N)", 저자·연도가 붙은 표,
    ENCODE/Human Protein Atlas 등 레퍼런스 데이터셋
- 애매하면 **cited로 보수적 처리**(frontmatter 오탐 방지). 태그는 self가 1건 이상일 때만 부여.

리뷰 논문은 대개 외부 데이터셋을 종합하므로 **cited-only**인 경우가 많다(태그 없이 섹션만).

## 절차

### 1) 후보 탐지 (기계적)
```bash
python3 tools/scan_accessions.py                 # extract/ 전체, 텍스트 리포트
python3 tools/scan_accessions.py --stem Foo2024   # 특정 노트만
python3 tools/scan_accessions.py --json           # 기계 판독용
```
노트별 블록의 accession을 ±문맥과 함께 출력한다(노트를 수정하지 않음, self/cited 판정 안 함).

### 2) self/cited 판정 (LLM)
각 accession을 위 신호구로 self/cited로 분류한다. OCR로 뭉개진 ID(예: 인접 참조번호가
붙어 `E-MTAB-1047626`)는 정상값으로 보정하거나 제외한다.

### 3) 노트 반영
frontmatter(self가 1건 이상일 때):
```yaml
tags: [YYYY-MM, <SubjectTag>, RawDataAvailable]   # 주제 태그 뒤에 속성 태그 추가
raw_data:                                          # self-deposited만, DB별 1줄
  - "GEO: GSE193742, GSE193743, GSE193744"
  - "PRIDE/ProteomeXchange: PXD010192"
```
본문 섹션(self·cited 모두 등장 시. Perspective 뒤, provenance footer 앞. 리뷰는 Key References 앞):
```markdown
## Data Availability

**Raw data generated by this study:**
- GEO: GSE193742, GSE193743, GSE193744

**Other accessions referenced in the text (reused/external data):**
- GEO: GSE118236
```
한국어 노트 헤더/라벨:
```markdown
## 데이터 이용 가능성

**본 연구가 생성·기탁한 원시 데이터:**
- GEO: GSE193742, GSE193743, GSE193744

**본문에 인용된 기타 accession (외부·재사용 데이터):**
- GEO: GSE118236
```
- 하위 소제목은 해당 항목이 있을 때만 넣는다(self 없으면 인용 항목만).
- accession run이 범위로 주어졌으면 범위 표기로 압축한다(예: `SRR16088842–SRR16088857`).

### 4) log + 미러
- `log`에 append: `"YYYY-MM-DD · edit · <Model> (<Tool>) · add RawDataAvailable tag, raw_data, Data Availability section"`
  (cited-only면 `· add Data Availability section`).
- 원저는 en/ko **양쪽** 노트에 동일하게 반영(accession ID는 언어 무관, 헤더/라벨만 언어별).

## 다른 KB에 적용
- 이 규약과 `scan_accessions.py`는 KB 비의존적이다. `--extract`로 대상 KB의 `extract/`를 지정한다.
- 주제 태그 체계와 무관하게 `RawDataAvailable`는 그대로 사용한다(속성 태그).
- extract 통합본이 없는 KB는 개별 추출 텍스트에서 같은 패턴으로 탐지하면 된다.
